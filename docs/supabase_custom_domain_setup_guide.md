# Инструкция по настройке Custom Domain для Supabase (MagicMusic CRM)

Данная настройка необходима для восстановления стабильной работы мобильного и ПК приложения MagicMusic CRM на территории РФ. Использование выделенного поддомена школы позволяет обойти региональные блокировки сетевых адресов Cloudflare и восстановить работу чатов (WebSockets).

---

## 1. Задача
Настроить поддомен (например, `api.magic-music.org`) так, чтобы он указывал на инфраструктуру Supabase. Это позволит приложению обращаться к базе данных через домен школы, который не имеет ограничений доступа.

## 2. Действия со стороны Администратора домена (DNS)
После активации функции в панели управления Supabase, необходимо будет добавить 2-3 записи в DNS-зону домена `magic-music.org`.

### Пример конфигурации (Mock-данные)
Ниже приведены примерные данные. Точные значения (токены) будут предоставлены после оплаты подписки на стороне Supabase.

| Тип записи | Имя (Host) | Значение (Value/Points to) | Примечание |
| :--- | :--- | :--- | :--- |
| **CNAME** | `api` | `xbnywnlhfgofskbdxb.supabase.co` | Основной шлюз API и БД |
| **TXT** | `_supabase.api` | `supabase-verification=xxxx-yyyy-zzzz` | Верификация владения доменом |

### Важное примечание для Cloudflare
Если домен `magic-music.org` обслуживается через Cloudflare, для данных записей **ОБЯЗАТЕЛЬНО** должен быть выбран режим **"DNS Only"** (серое облако). Режим "Proxied" (оранжевое облако) может конфликтовать с SSL-сертификатами, которые Supabase выпускает автоматически.

---

## 3. Порядок действий

1.  **Согласование**: Администратор подтверждает возможность создания поддомена.
2.  **Активация**: Разработчик приложения активирует Pro-план и функцию Custom Domain в Supabase.
3.  **Передача данных**: Разработчик отправляет администратору финальные значения `Value` и `Token`.
4.  **Обновление DNS**: Администратор вносит записи. Время распространения DNS обычно составляет от 15 минут до пары часов.
5.  **Верификация**: Supabase автоматически проверяет записи и выпускает SSL-сертификат.
6.  **Запуск**: Разработчик переключает приложение на новый URL.
