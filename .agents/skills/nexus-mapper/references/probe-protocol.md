# Протокол PROBE — Детальное руководство по этапам

> Этот документ является исполнительным планом для SKILL.md. Первым шагом после активации навыка является полное прочтение этого файла. Перед этапом EMIT также необходимо изучить `references/output-schema.md`.

---

## P — PROFILE (Профилирование)

**Предварительная проверка**
1. Подтвердите существование `$repo_path`.
2. Проверьте наличие `$repo_path/.git`.
   - Есть: выполнить анализ тепловых карт Git.
   - Нет: зафиксировать `git analysis skipped`, продолжить анализ AST и дерева файлов.

**Шаги выполнения**
1. Запуск экстрактора AST (и генерация отфильтрованного дерева файлов).
2. Запуск анализатора Git (если есть `.git`).

**Чек-лист завершения**
- [ ] `raw/ast_nodes.json` записан.
- [ ] `raw/file_tree.txt` не пуст.
- [ ] Если есть Git: `raw/git_stats.json` содержит данные о «горячих точках» (hotspots).

---

## R — REASON (Рассуждение)

**Проверка граничных сценариев**
- Новый репозиторий (1 коммит): пропустить анализ Git.
- Монорепозиторий (>1000 файлов): использовать ограничение узлов (`--max-nodes 200`).
- Нет README: искать информацию в конфигах (`pyproject.toml`, `package.json`).

**Стратегия чтения (приоритет)**
1. `README.md` — общее описание.
2. Файлы зависимостей — стек и библиотеки.
3. Точки входа (`main.py`, `index.ts`).
4. `raw/file_tree.txt` — структура папок.

---

## O — OBJECT (Объективизация/Сомнение)

**Зачем это нужно**: Первое впечатление часто обманчиво (названия папок не всегда отражают суть, импорты могут быть запутаны).

**Протокол сомнения**: Сформулируйте 1–3 гипотезы, которые ставят под вопрос ваше текущее понимание архитектуры.
- *Пример*: «Git показывает высокую активность в `tasks/`, но я считал точкой входа `evolution/`. Почему?»

---

## B — BENCHMARK (Проверка гипотез)

Для каждого «сомнения» из этапа OBJECT:
1. Найдите доказательства через `grep_search` или `view_file`.
2. Проверьте реальные зависимости через `query_graph.py --impact`.
3. Скорректируйте или подтвердите архитектурную модель.

---

## E — EMIT (Вывод)

> [!IMPORTANT]
> **Ограничение этапа**: Перед записью ЛЮБОГО файла вы ОБЯЗАНЫ прочитать `references/output-schema.md`.

**Порядок записи (сначала в `.tmp/`, затем перенос)**
1. `concepts/concept_model.json` (Схема V1).
2. `INDEX.md` (L0 cводка, < 2000 токенов).
3. `arch/systems.md` (Границы систем).
4. `arch/dependencies.md` (Диаграммы Mermaid).
... и остальные файлы.

**Требования к INDEX.md**
- Объем < 2000 токенов.
- Конкретные выводы, отсутствие «воды».
- Обязательный блок «Инструкция по эксплуатации» для будущих сессий ИИ.
