# Шаблон ADR (Architecture Decision Record)

## Назначение
Этот шаблон используется для фиксации ключевых архитектурных решений. Каждый ADR должен быть посвящен **одному решению**.

---

---

## Лучшие практики

1. **Краткость**: ADR должен читаться за несколько минут.
2. **Фокус на одном вопросе**: Сложные решения разбивайте на несколько ADR.
3. **Неизменность**: Принятое решение — это исторический факт. Для изменений создавайте новый ADR.
4. **Версионность**: Храните ADR вместе с кодом в Git.
5. **Влияние**: Всегда указывайте системы, на которые влияет это решение.
